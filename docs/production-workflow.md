# 完整制作流程（从 0 到可运营）

下文按 **阶段** 组织，每阶段列出目标、主要活动、交付物与验收标准，便于排期与分工。

---

## 阶段 0：立项与范围

**目标**：明确「超长」的量化指标与产品边界，避免后期架构返工。

**活动**

- 定义成功指标：例如单部作品目标总字数、更新频率、同时在线作品数、峰值并发生成任务数。
- 划分用户角色：作者（人类主导）、协作者、读者、运营/审核。
- 明确合规：内容政策、未成年人保护、版权与训练数据声明（若涉及用户上传语料）。

**交付物**

- 产品需求说明（PRD）一页纸 + 用户故事地图。
- 非功能需求（NFR）：可用性、延迟上限、数据保留周期、导出格式。

**验收**

- 干系人对「超长」定义与合规边界签字或书面确认。

---

## 阶段 1：概念验证（PoC）

**目标**：用最小代码验证「分章 + 记忆摘要 + 续写」能否在 **3～5 章** 内保持人物与设定不崩。

**活动**

- 选定 1～2 个模型 API（或本地推理），实现最小链路：大纲 → 第 1 章 → 摘要 → 第 2 章 …
- 手工检查：时间线、称谓、关键道具是否一致。

**交付物**

- 可运行的 PoC（可为脚本或单体服务）。
- PoC 报告：失败案例类型（遗忘、自相矛盾、风格漂移）。

**验收**

- 连续章节人工评分达到约定阈值，或明确列出必须工程化的缺陷清单。

---

## 阶段 2：领域建模与数据设计

**目标**：把「小说」从纯文本提升为 **可版本化、可检索、可续写** 的结构化实体。

**活动**

- 设计核心实体：作品（Work）、卷（Volume）、章（Chapter）、场景（Scene）、角色（Character）、地点（Location）、时间线事件（TimelineEvent）、设定条目（CanonEntry）、记忆块（MemoryChunk）、生成任务（GenerationJob）。
- 设计状态机：章节从草稿 → 待审 → 已发布；任务从排队 → 运行 → 成功/失败/取消。
- 设计幂等与重试：同一任务重复提交不产生重复章节。

**交付物**

- ER 图或等价 schema 说明。
- 迁移策略（若使用关系库）：初始 migration 与回滚策略。

**验收**

- 任一实体可独立 CRUD，且外键与级联删除策略文档化。

---

## 阶段 3：Go 服务骨架与 API

**目标**：建立可扩展的单体或多服务边界，接口稳定后再填业务。

**活动**

- 初始化 Go module、分层目录（见 `golang-architecture.md`）。
- 定义 HTTP/gRPC 契约：作品管理、章节读写、任务提交、任务查询、导出。
- 接入配置、日志、追踪 ID、健康检查。

**交付物**

- OpenAPI 或 protobuf 定义。
- 本地 `docker compose` 一键起依赖（数据库、队列、可选向量库）。

**验收**

- 契约测试或 smoke 脚本通过；CI 可编译与跑单元测试。

---

## 阶段 4：长篇小说引擎 MVP

**目标**：实现 **一章一章** 的稳定生成，带摘要与简单检索。

**活动**

- 实现流水线：规划（可选）→ 写章 → 抽取结构化更新 → 写入记忆 → 校验。
- 实现「上一章摘要 + 本书大纲片段 + 本章写作提示」的上下文拼装。
- 失败重试与部分成功处理（例如写成功但摘要失败）。

**交付物**

- 引擎配置项：温度、最大 token、重试次数、摘要模型与写作模型是否分离。

**验收**

- 自动化回归：固定种子或 fixture 下，同一输入产生可接受的输出波动范围（若模型非确定性，则验结构而非逐字）。

---

## 阶段 5：超长增强（向量检索 + 细粒度记忆）

**目标**：当章节数超过「纯摘要」可承载上限时，仍能通过检索命中早期细节。

**活动**

- 对章节/场景做分块 embedding，写入向量库；检索 query 由「本章目标 + 涉及角色/地点」拼装。
- 引入 **分层记忆**：全书摘要、卷摘要、章摘要、原文片段引用（带出处）。
- 冲突检测：新文本与 Canon 矛盾时标记或触发重写。

**交付物**

- 检索命中率与延迟监控面板说明。
- 调参文档：topK、重排序、混合检索（关键词 + 向量）。

**验收**

- 在长 fixture（例如 50 章模拟数据）上，针对预设「伏笔问题」检索与生成能命中正确前文。

---

## 阶段 6：人机协同与工作流

**目标**：平台不仅自动生成，还支持人类精修与分支管理。

**活动**

- 差异对比（diff）、批注、锁定段落（模型不可覆盖）、版本回滚。
- 「从某一章重生成」时，后续章节是否级联失效的策略（建议：标记下游为 stale）。

**交付物**

- 前端或 CLI 工作流说明（可与现有 Web 仓库对接）。

**验收**

- 典型用户故事：锁一段对话 → 重生成其余 → 导出全书 Markdown/EPUB。

---

## 阶段 7：质量、安全与运营

**目标**：可控成本、可审计、可应急。

**活动**

- 内容审核（规则 + 模型分类器）、速率限制、滥用检测。
- 成本核算：按字数/按 token/按任务计费与预算告警。
- 备份与灾难恢复演练。

**交付物**

- 运行手册、告警规则、on-call 流程。

**验收**

- 桌面演练一次：主库故障切换或队列堆积恢复。

---

## 阶段 8：发布与迭代

**目标**：灰度上线，数据驱动迭代。

**活动**

- 特性开关：新检索策略、新模型可灰度。
- 收集结构化反馈：段落级点赞/问题标签。

**交付物**

- 发布清单（checklist）与回滚方案。

**验收**

- 生产环境稳定运行约定时长且无 P0 未关闭事件。

---

## 建议排期（仅供参考）

| 阶段 | 大致周期（小团队） |
|------|-------------------|
| 0～1 | 1～2 周 |
| 2～3 | 2～4 周 |
| 4 | 3～6 周 |
| 5 | 4～8 周 |
| 6～8 | 持续迭代 |

实际周期随模型质量、是否自托管推理、合规要求强弱而显著变化。
