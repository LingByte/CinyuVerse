# 质量、安全、成本与运维

超长篇小说平台上线后，问题往往出在 **成本爆炸、内容合规、不可复现的生成缺陷**。本章给出可操作的工程化对策。

---

## 1. 质量保障体系

**分层测试**

- **单元测试**：状态机迁移、幂等键、摘要拼接逻辑、时间线单调性。
- **合同测试**：对外 API 契约稳定。
- **回归集**：固定世界观 fixture + 自动生成 N 章 + 自动断言（无死锁、无空章、schema 合法）。

**人工评测**

- 按「开篇 / 中段 / 高潮 / 卷末」抽样，使用统一评分表：连贯性、人物一致性、节奏、文笔、设定遵守。

**线上观测**

- 记录每章：`prompt_version`、`model`、`input_tokens`、`output_tokens`、检索 hit 列表摘要（非全文）、校验失败次数。

---

## 2. 内容安全与合规

**多层防护**

1. **输入侧**：用户提示长度与模式过滤；禁止明显违法指令（需法务定稿）。
2. **输出侧**：分类模型或第三方审核 API；命中则阻断发布或仅保存为草稿并提示修改建议。
3. **运营侧**：举报通道、人工复核队列、账号处罚策略。

**审计**

- 对审核决策与人工修改保留日志：谁、何时、改了什么字段。

---

## 3. 成本与配额

**计费维度**

- 按 token、按章节任务、或按订阅档位包月配额。

**优化手段**

- 摘要用小模型；检索 topK 控制；缓存重复查询的 embedding。
- 对「重写」设每日上限；对免费用户降采样检索或缩短上下文。

**预算告警**

- 按模型供应商账单 + 自建 GPU 利用率设阈值；异常飙升触发自动限流。

---

## 4. 可靠性与灾备

**数据**

- PostgreSQL 定期快照 + WAL 归档（按云厂商最佳实践）。
- 向量库定期备份或可从原文重建（重建成本高，需 playbook）。

**队列**

- 死信队列（DLQ）与人工重放工具；任务超时自动取消并释放配额。

**多副本**

- API 无状态水平扩展；Worker 根据队列深度 HPA。

---

## 5. 可观测性

**指标（示例）**

- `job_duration_seconds`（histogram）
- `llm_calls_total`、`llm_errors_total`
- `retrieval_hits`（histogram of topK scores）

**追踪**

- 将 `trace_id` 从 HTTP 传入 engine、LLM 客户端，便于跨服务查慢请求。

---

## 6. 用户数据与版权

- 用户作品默认归属需在 ToS 中写明；导出格式（Markdown、EPUB、DOCX）应可离线保存。
- 若使用用户文本训练平台模型，需 **单独明示同意**（视司法辖区而定）。

---

## 7. 发布与回滚

- **特性开关**：新检索策略、新模型、新校验器可灰度到部分作品或部分用户。
- **回滚**：保留 `prompt` 与 `model` 版本；出现质量回归时一键切回上一稳定组合。

---
